01

Install & First Run

1
Download the .zip

Get glacier2_007_toolkit.zip from the Download page. Do not extract it.

2
Open Blender Preferences

Edit → Preferences → Add-ons → Install from Disk…

3
Select the zip

Navigate to glacier2_007_toolkit.zip and click Install Add-on.

4
Enable the addon

Search "Glacier" in the addon list and tick the checkbox. Click Save Preferences.

5
Open the panel

In the 3D Viewport, press N to open the sidebar and select the Glacier tab.

Requires Blender 4.2 or later. The panel will not appear in older versions.
02

The Panel

All toolkit operations live in the N-panel → Glacier tab. The panel is context-sensitive — some buttons only appear when a relevant object is selected.

Panel Sections

Hover over any button to see a tooltip describing what it does and what selection it expects.
03

Import a .prim

The importer reads the binary .prim format, extracts all objects (meshes, LODs, cloth), reconstructs the armature from the linked .borg, and brings everything into Blender as a collection.

1
Extract game files

Use RPKG-Tool to extract the .prim and its linked .borg (skeleton). They must be in the same folder.

2
Import

Panel → Import Prim. Navigate to the .prim file. The .borg is loaded automatically if it's in the same directory.

3
Check the collection

A new collection appears in the outliner named after the .prim. It contains the armature, all mesh objects, and cloth cages.

Import Options

Import ClothOnImports cloth cage meshes as separate objects
Auto Apply MaterialOnLoads and applies .MATI material on import
Load Render MaterialOffBuilds the full Cycles/EEVEE node tree with textures
04

Materials

The Glacier 2 material system uses .MATI (material instance) files that reference textures and a shader class. The toolkit reads and writes these files natively.

Load a material onto a mesh

1
Select the mesh

Click the mesh object in the viewport.

2
Load Material

Panel → Materials → Load .MATI. Navigate to the extracted .MATI file.

3
Apply

The material is assigned to the mesh and its texture slots are populated from the linked .TEXT/.TEXD files.

The toolkit resolves texture paths relative to the .MATI file. Keep all extracted files in the same folder.
05

Render Materials

Render materials build a full Blender node tree for Cycles or EEVEE — diffuse, normal, SRM (Specular/Roughness/Metallic), and emissive — decoded directly from .TEXT/.TEXD assets.

Build the render material

Select a mesh that already has a .MATI loaded, then: Panel → Materials → Build Render Material. This decodes all referenced textures and wires them into a PBR node setup.

06

Render Source

The Render Source system links a .prim to a specific RPKG chunk file so the toolkit knows where to write the output. Set this before exporting.

1
Select the prim root

Click the armature or root collection object for your prim.

2
Set Render Source

Panel → Export → Set Render Source. Navigate to the source .rpkg chunk that contained the original .prim.

3
Confirm

The chunk path is stored on the object. The exporter uses it to generate correct hash filenames and the patch RPKG metadata.

07

Edit a Material

You can change any texture slot in a loaded material — swap in a new diffuse, normal, or SRM texture — and then export the modified .MATI and textures as part of your patch.

Swap a texture slot

1
Open Material Properties

Select the mesh → Properties panel → Material Properties. Find the slot you want to replace.

2
Assign a new image

Click the image field and pick or load your new texture. PNG and TGA are both supported.

3
Encode on export

The exporter will encode your texture to the correct BC format automatically when you run Export Prim.

Match the BC format of the original slot. Swapping a BC1 diffuse with a BC3 source gives you full alpha. Mismatched formats can cause in-game artifacts.
08

Swap a Material

Material swap replaces which .MATI a mesh object references — useful for applying a completely different material from another character or prop.

1
Select the mesh

The mesh must already have a material loaded via the toolkit.

2
Swap Material

Panel → Materials → Swap Material. Navigate to the replacement .MATI file.

3
Rebuild render material

Run Build Render Material again to see the new textures in the viewport.

09

Texture Tools

The Texture Tools section handles the Glacier 2 texture format (.TEXT / .TEXD) — decode to editable images and encode your edits back.

Decode .TEXT / .TEXD to image

1
Decode Texture

Panel → Texture Tools → Decode .TEXT. Pick the .TEXT file. The .TEXD is loaded automatically from the same folder.

2
Result

A PNG (or TGA for normal maps) is written next to the source file and opened in Blender's Image Editor.

Encode image to .TEXT / .TEXD

1
Edit the image

Paint or replace the decoded PNG in any image editor.

2
Encode Texture

Panel → Texture Tools → Encode .TEXT. Pick the edited PNG. The toolkit compresses to the correct BC format and writes .TEXT + .TEXD.

10

LOD Levels

Glacier 2 meshes can have up to four LOD levels (LOD0 – LOD3). LOD0 is the full-detail mesh used at close range. Higher LODs are simplified meshes for distance culling.

Only LOD0 is imported by default. Enable Import LODs in the import options to bring in all levels.

Editing LODs

Each LOD is a separate mesh object in the collection. You can edit or replace individual LODs — the exporter writes all LODs present in the collection.

If you want a custom mesh swap but don't want to touch LODs, edit only LOD0. The game will just use the original (unchanged) lower LODs at distance, which is usually fine.
11

Edit the Mesh

UPDATED v2.9.3

You can reshape, sculpt, or fully replace any imported mesh. The toolkit supports two workflows depending on whether you need to change the vertex count.

Option A — Move existing vertices (safest)

Use Sculpt Mode, Edit Mode, or any Blender modifier that doesn't add or remove vertices. The exporter byte-patches positions in-place — the skin partition, bone remap, and aux cluster data are copied verbatim. This is 100% reliable.

Prefer this path for cloth cages. Cloth link data is vertex-index based — adding or removing vertices breaks the cloth simulation.

Option B — Custom topology (different vertex count)

As of v2.9.3, exporting a skinned mesh with a different vertex count is fixed. The bone-remap table that previously caused in-game vertex explosions now uses a correct identity mapping for each bone actually used by the mesh.

The aux cull-cluster block can't be regenerated from scratch. The toolkit writes an empty cluster block (type 0x20, 0 entries). This may affect very large distance culling — for most characters it's imperceptible.
1
Transfer weights

Select your new mesh, then shift-click the imported game mesh (making it active). Panel → Mesh Tools → Transfer Weights to Selected.

2
Make exportable

With your new mesh selected: Panel → Mesh Tools → Make Custom Mesh Exportable. This copies the required Glacier metadata from the game mesh.

3
Export with Custom Mesh on

Panel → Export → enable Experimental: Custom Mesh → Export Prim.

For an even safer path that keeps the original vertex count, see §16 Custom Model Swap — Method B.

12

Export

The exporter writes a new .prim file and — optionally — packs everything into a patch .rpkg that can be dropped directly into the game's Runtime folder.

1
Select the root object

Click the armature or the root collection object for the prim you want to export.

2
Set export path

Panel → Export → Output Folder. Pick a folder for the output files.

3
Configure options

Enable Pack RPKG to auto-generate a patch archive. Enable Custom Mesh only if you're exporting a mesh with a different vertex count.

4
Export Prim

Panel → Export Prim. The .prim is written, textures are encoded, and — if enabled — a patch_xxx.rpkg is generated in the output folder.

5
Deploy

Copy the patch_xxx.rpkg to: [Game Install]\Runtime\. Launch the game.

Export Options

OptionDefaultEffect
Pack RPKGOffWraps output into a deployable patch archive
Experimental: Custom MeshOffFull prim rebuild — required for changed vertex counts
Export TexturesOnEncodes and includes modified textures in the output
Export MaterialsOnWrites modified .MATI files to the output

Full Walkthrough

End-to-end walkthrough: replace Bond's jacket with a custom model and export a deployable patch.

1
Extract the asset

Open RPKG-Tool → Extract → find Bond's jacket .prim and its .borg, .MATI, .TEXT, .TEXD. Save to a work folder.

2
Import

Blender → Glacier panel → Import Prim. Enable Load Render Material so you see the textured result in the viewport.

3
Edit the mesh

Sculpt or replace as needed. For a custom model, follow §16.

4
Edit textures (optional)

Decode the diffuse .TEXT to PNG, repaint it, encode it back.

5
Set Render Source

Select the armature → Panel → Export → Set Render Source → pick the original .rpkg chunk. This lets the toolkit generate correct hash names.

6
Export

Panel → Export → enable Pack RPKG → Export Prim. A patch_xxx.rpkg appears in your output folder.

7
Deploy

Copy patch_xxx.rpkg to [Game]\Runtime\. Launch 007 First Light and check your mod in-game.

16

Custom Model Swap

v2.9.3 NEW

You can replace any game mesh with your own model — a completely different shape, topology, and vertex count — and have it run in-game with rigging, UVs, and materials intact.

There are two methods. Method A allows a completely different vertex count. Method B is the safest path and keeps the original vertex count by conforming the game mesh to your custom shape.

Method A — Custom Topology

Use this when your custom mesh has a different vertex count from the original. This triggers a full PRIM rebuild. Fixed in v2.9.3 — the bone-remap bug that caused in-game vertex explosions is corrected.

Method A steps

Import your custom mesh into the same Blender scene as the game's prim.

1
Import game prim

Panel → Import Prim. This gives you the game mesh with its weights and skeleton.

2
Import your custom model

File → Import → your preferred format (OBJ, FBX, etc.). Position it to overlap the game mesh.

3
Transfer Weights

Select your custom mesh → shift-click the game mesh (active). Panel → Mesh Tools → Transfer Weights to Selected.

4
Make Exportable

Select your custom mesh. Panel → Mesh Tools → Make Custom Mesh Exportable. This copies Glacier metadata from the game mesh to your object.

5
Hide / delete the game mesh

The game mesh is no longer needed in the export. Hide or delete it from the collection.

6
Export with Custom Mesh

Panel → Export → enable Experimental: Custom Mesh → Export Prim.

If the mesh still looks wrong in-game after updating to v2.9.3, double-check that weights were transferred before making the mesh exportable, and that all vertex groups belong to the correct armature bones.

Method B — Snap Game Mesh to Custom Shape

Use this when you want your custom model's silhouette but want to keep the exact original vertex count. The game mesh's vertices are projected onto the surface of your custom model using Shrinkwrap — the rigging is completely untouched.

Method B steps

1
Import game prim

Panel → Import Prim.

2
Import your custom model

File → Import your model. Position and scale it to match the game mesh proportionally.

3
Select — active order matters

Click your custom mesh first, then shift-click the game mesh to make it the active object.

4
Snap Game Mesh to Custom Shape

Panel → Mesh Tools → Snap Game Mesh to Custom Shape. Optionally enable Transfer UVs from Custom Mesh to also copy your custom UVs.

5
Make Exportable

Panel → Mesh Tools → Make Custom Mesh Exportable.

6
Export normally

No need to enable Custom Mesh — the vertex count is the same as the original. Just run Export Prim.

Method B is the safest export path. Because the vertex count is unchanged, the safe-path byte-patcher runs — skin partition, BoneInfo, BoneIndices, and aux cluster are all copied verbatim from the original.

Choosing between the two methods

Method A — Custom TopologyMethod B — Snap
Vertex countAny (can change)Same as original
TopologyFully customOriginal topology, new positions
RiggingTransferred (requires step)Original — untouched
UVCustom UVs keptOriginal or transferred
Export pathFull PRIM rebuildByte-patch (safest)
Cloth supportDrop cloth cagesCloth intact
Aux clustersEmpty block (minor cull impact)Original block kept
