The Pipeline

FROM GAME FILES TO IN-GAME RESULT

Every mod follows the same four-step pipeline. The toolkit handles each step inside Blender — no external converters mid-flow.

01

Extract

Use RPKG-Tool to pull .prim, .borg, .MATI, and texture files from the game's chunk archives.

02

Import

Load the files into Blender. Auto-Apply builds the full render material — textures, normals, SRM — on import.

03

Edit

Reshape the mesh, repaint textures, swap materials, or replace the model entirely with your own.

04

Export & Pack

Export the .prim, then pack everything into a patch .rpkg straight from the sidebar panel.

v2.9.3 — FIXED & NEW

CUSTOM MODEL SWAP

Import your own mesh — different vertex count, different topology — and get it running in-game with rigging and UVs intact. The bone-remap explosion bug is fixed. Two methods, one goal.

See the Guide
Method A — Custom Topology (Fixed in v2.9.3)

Transfer weights to your mesh, make it exportable, export with Custom Mesh enabled. The all-0xFF bone-remap that caused vertex explosions in-game is corrected — your mesh now uses a proper identity remap.

Method B — Snap to Game Slot (New in v2.9.3)

Projects the original game mesh's vertices onto your custom model's surface via Shrinkwrap. Keeps the exact original vertex count and rigging intact — the safest export path, no Custom Mesh flag required.
